## 🔬 技巧 3：First Principles Thinking（第一原理）

**目標：** 從基礎原理重建技術架構，確保可行性

### 核心洞察

#### 第一原理分析

**原理 #1：光的本質**
- 光 = 顏色 + 透明度 + 位置
- Flutter CustomPainter + AnimationController 完全足夠
- 不需要複雜 3D 引擎或外部依賴

**原理 #2：3D 空間感**
- 3D 感 = 視覺錯覺
- Stack + Transform + Opacity 實現偽 3D
- 比真 3D 更輕量、更快

**原理 #3：呼吸光流**
- 呼吸 = sin/cos 曲線
- 光流 = Path + Gradient + Animation
- 核心邏輯不到 50 行代碼

**原理 #4：粒子系統**
- 粒子 = {x, y, vx, vy, alpha, size}
- 對象池 + 空間分區 + 批次繪製
- 500-1000 粒子在 60fps 完全可行

**原理 #5：滑動手勢**
- Flutter GestureDetector 原生完善
- 精確計算：力道 = 速度權重(0.6) + 距離權重(0.4)

### 技術架構決策

**核心系統（3 個）：**

```
1. 渲染引擎（Rendering Core）
   ├─ CustomPainter（主繪製器）
   ├─ 粒子系統管理器
   ├─ 光流路徑生成器
   └─ 節拍波紋生成器

2. 動畫調度器（Animation Orchestrator）
   ├─ 單一 Ticker（統一時間源）
   ├─ 呼吸曲線生成
   ├─ 過渡動畫管理
   └─ 60fps 保證機制

3. 狀態管理器（State Manager）
   ├─ 生命值狀態
   ├─ 粒子狀態
   ├─ 藍牙連接狀態
   └─ UI 模式狀態
```

**效能策略：**

```
目標：
✅ 60fps 穩定（16.67ms/幀）
✅ 低電量消耗
✅ 快速啟動（<1秒）
✅ 記憶體穩定（<100MB）

方法：
├─ 單一 AnimationController（避免多 ticker）
├─ 批次繪製（減少 draw calls）
├─ 對象池（減少 GC）
├─ 條件渲染（只繪製可見元素）
└─ 硬體加速（RepaintBoundary）
```

**技術選擇：**

| 需求 | 複雜方案 | 第一原理方案 | 選擇 |
|------|----------|--------------|------|
| 光效 | Flame 引擎 | CustomPainter | ✅ 簡單 |
| 3D感 | three_dart | Stack+Transform | ✅ 簡單 |
| 粒子 | 物理引擎 | 手寫輕量系統 | ✅ 簡單 |
| 狀態 | Redux | Provider/Riverpod | ✅ 簡單 |
| 藍牙 | 多個套件 | flutter_blue_plus | ✅ 簡單 |

**開發時程估算：**

```

├─ 生命值顯示 + 基礎動畫
├─ 滑動輸入
└─ 簡單光流效果

Phase 2（視覺增強）：2 週
├─ 粒子系統
├─ 呼吸效果
└─ 3D 空間感

Phase 3（功能擴充）：1-2 週
├─ 骰子/硬幣
├─ 計數器
└─ 節拍視覺化
```
